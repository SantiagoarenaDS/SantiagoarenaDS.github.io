Pipeline SLI: detección del trastorno específico del lenguaje

En agosto de 2024 presenté en JAIIO un pipeline para detectar Trastorno Específico del Lenguaje (SLI) a partir de transcripciones de narrativas espontáneas. La presentación fue realizada junto a Ricardo Di Pasquale, director de la carrera de Ciencia de Datos de la UCA y ML Engineering Associate Director en Accenture.

Problema

El proyecto busca apoyar la detección del SLI usando variables extraídas de narrativas espontáneas. La hipótesis técnica es que indicadores lingüísticos como morfemas, errores de palabra, proporción de verbos y riqueza léxica pueden aportar señal para separar casos de desarrollo típico y SLI.

Pipeline

  • Carga de datos lingüísticos desde `all_data_R.csv`.
  • Limpieza de vacíos, remoción de duplicados y normalización de variables categóricas.
  • Selección de variables mediante Random Forest, matriz de correlación Spearman e importancia Gini.
  • Regresiones logísticas iterativas para retener variables significativas y explicables.
  • Modelo KNN con partición train/test y validación cruzada para seleccionar `k`.
  • Evaluación con matriz de confusión, pérdida cero-uno, curva ROC y AUC.

Variables clave

El análisis priorizó variables como `r_2_i_verbs`, `mlu_morphemes`, `word_errors`, `regular_past_ed`, `average_syl` y `freq_ttr`. El proyecto no solo optimiza performance: también cuida que las variables seleccionadas puedan ser interpretadas por usuarios clínicos o investigadores del lenguaje.

Presentación

La presentación en JAIIO 2024 fue una instancia de comunicación técnica: llevar un pipeline reproducible desde el código hacia una audiencia académica, explicar decisiones de modelado y mostrar cómo la ciencia de datos puede dialogar con problemas de lenguaje y salud.
