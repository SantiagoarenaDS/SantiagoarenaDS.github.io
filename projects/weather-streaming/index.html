Tablero Climático Kafka: producto de streaming meteorológico en Colab

Notebook autocontenido que convierte un dashboard climático interactivo en un producto de ingeniería de datos con ingesta, enriquecimiento y mensajería en tiempo real. El flujo levanta ZooKeeper y Kafka dentro de Colab, publica snapshots de Open-Meteo en un topic y consume esos eventos para validar una tabla enriquecida antes de renderizar el tablero HTML original.

Arquitectura

El producto fusiona dos piezas: el tablero climático con mapa y el pipeline compacto de Kafka para Buenos Aires. La solución instala Java, Kafka 3.9, ZooKeeper y dependencias Python; luego arranca los servicios, crea el topic `tablero-climatico-ciudades` y ejecuta un ciclo producer-consumer verificable.

La capa de streaming enriquece cada ciudad con variables meteorológicas y calidad de aire: temperatura, sensación térmica, humedad, lluvia, viento, ráfagas, presión, UV, PM2.5, PM10 y AQI europeo. Esa salida alimenta una tabla Pandas auditable y deja el tablero HTML final intacto.

Flujo de datos

  • Open-Meteo Forecast API y Air Quality API proveen snapshots actuales por ciudad.
  • Un productor Python serializa cada medición como evento JSON y la publica en Kafka.
  • ZooKeeper coordina el broker para mantener el modo clásico de Kafka solicitado.
  • Un consumidor lee desde el topic, toma el último evento por ciudad y valida cobertura completa.
  • El notebook muestra una tabla enriquecida y luego renderiza el dashboard interactivo original.

Decisión técnica

La implementación mantiene el dashboard como superficie estable: no se reescribe el HTML final ni se cambia su contrato visual. Kafka queda como capa de ingeniería de datos alrededor del tablero, separando ingesta, transporte, consumo y visualización.

Resultado: un demo ejecutable en Colab que prueba streaming local con Kafka/ZooKeeper y conserva el producto analítico para usuarios finales.
